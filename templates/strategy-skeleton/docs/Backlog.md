# Бэклог (триггерные записи)

> **Назначение:** записи `B-NNN`, которые НЕ становятся РП сейчас, но должны быть
> поднятыми, когда сработает явный триггер. Не путать с «Бэклог» в Strategy.md
> (тот — общий список направлений без триггеров).

## Формат записи

Каждая запись — один блок вида:

```
### B-NNN: Название (существительное-артефакт)

**Дата открытия:** YYYY-MM-DD
**Триггер открытия:**
- Hard-trigger: <внешнее событие, которое должно случиться буквально>
  (например: «первый user-deletion request получен», «legal review запланирован на эту неделю»)
- Soft-trigger: <дата/процессная веха>
  (например: «при открытии WP-XXX-v2», «при следующей ревизии DP.D.NNN»)

<1-3 предложения контекста — зачем эта запись, что произойдёт когда триггер сработает>
```
